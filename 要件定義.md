# 1. はじめに

このアプリケーションは、会議の議事録を作成するためのMacで動くものです
ステータスメニューに常駐して、いつでも録画開始ボタンを押すことができ、録画が終わったら自動的にウィスパーを使って文字起こしをしてLLMで会議の要約を作ってくれます

# 2. 背景・課題

開発の背景としては、会議が終わった後に会議の内容を見返せるようにしたいと言う意図があります。ただし、既存のツールではクラウド連携が前提となっており使用しづらいです。このツールではローカルで動くLLMを活用することによって、セキュアに文字起こしをしたいと考えています。

# 3. 利用者・利用シーン

想定利用者としては、Macを使用している会議に参加しているユーザです。利用シーンとしては、Teams会議やZoom会議やどんな会議でも使用したいと思っています。

# 4. 機能要件

必要な機能としては、まずは画面録画を開始できる機能、画面録画を終了できる機能、それから録画が終わった際に自動的に文字起こしが走る機能、そして文字起こしが終わった後に会議の要約を作成できる機能が欲しいです。そしてそれらは自動的にすべて行って欲しいです。

# 5. 非機能要件

非機能要件としては、よく考えられたアーキテクチャを採用し、後でコードを見直したとしても、簡単に修正できるようなディレクトリー構造やコードの書き方になっていることが望ましいです
アプリケーションは、Macが立ち上がったときに自動的に立ち上がるようにしてください

# 6. 画面・UI要件

インストールしたアプリケーションをクリックすると、開く画面
- 文字起こしに利用するモデルや会議の予約を行うモデルを選択できるようにしてください
- 録画と文字起こしと要約が終わった際に、それらを出力するフォルダを選択できるようにしてください

ステータスメニューをクリック際に開く画面
- 録画を開始するウィンドウを選択できるようにしてください。そしてその選択したウインドウの音声もキャプチャーするようにしてください。
- もしウィンドを選択しない場合は、デフォルトで画面全体の録画と録音をするようにしてください
- 録画を開始と終了するボタンを置いてください

# 7. システム構成

ウィスパーやローカルLLMモデルはアプリケーション自体には含めずに追加ダウンロードできるようにしてください

# 8. 移行・導入計画

AIを用いた開発をしたいので、適したときに適したプラグインやスキル、フックなどを導入できるようにしたいです
もしAIの方で、この作業は、事前にプラグインやスキルを入れた方が良いと判断された場合は、実行前に推奨してください。
例えば、環境の構築の時などがそれに当たります。特にライブラリの入や環境の構築の際には、公式ドキュメントなどを参考したりして想像で実装しないようにしてください。
