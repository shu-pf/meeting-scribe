# 初回起動フローのテスト手順

## 前提

- アプリを**一度でも起動してガイダンスを閉じている**と、`hasSeenFirstLaunchGuidance` が `true` になり、初回フローは再現しません。
- テストするたびに「初回状態」に戻す必要があります。

## 1. 初回状態に戻す

ターミナルで次を実行し、ガイダンス未読状態にします。

```bash
defaults delete com.shu-pf.MeetingScribe hasSeenFirstLaunchGuidance
```

（オプション）設定を**すべて**リセットしたい場合（Whisper モデル選択なども含む）:

```bash
defaults delete com.shu-pf.MeetingScribe
```

## 2. アプリを起動する

1. アプリが起動中なら**終了**する。
2. Xcode から Run（⌘R）するか、ビルド済みの MeetingScribe を起動する。

## 3. 初回フローを確認する

1. **メニューバー**の MeetingScribe アイコン（四角いメニューバーアイコン）をクリックする。
2. 期待動作:
   - **設定ウィンドウが自動で開く**
   - **「はじめての方へ」ガイダンスのシートが表示される**（Whisper のダウンロード画面はまだ出ない）
3. ガイダンスで **「始める」** をクリックする。
4. 期待動作:
   - ガイダンスが閉じる
   - **そのあと**、Whisper モデルが未設定なら **Whisper モデルダウンロード画面のシート**が表示される

## 4. 2回目以降の挙動を確認する（任意）

1. 上記のとおり一度ガイダンスを閉じた状態にする。
2. アプリを終了せず、もう一度メニューバーをクリックする。
3. 期待動作:
   - 設定ウィンドウは**自動では開かない**（「設定を開く」を押すと開く）。
4. 「設定を開く」で設定を開く。
5. 期待動作:
   - ガイダンスは出ない。Whisper 未設定のときだけダウンロード画面が出る。

## チェックリスト

- [ ] `defaults delete` 後に起動し、メニューを開くと設定が自動で開く
- [ ] 初回はガイダンスだけ表示され、Whisper ダウンロード画面は被らない
- [ ] 「始める」を押すとガイダンスが閉じ、そのあと Whisper ダウンロード画面が出る（モデル未設定時）
- [ ] 2回目はメニューを開いても設定は自動で開かず、「設定を開く」で開ける
