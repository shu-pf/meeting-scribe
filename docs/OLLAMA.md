# Ollama 連携（要約機能）

MeetingScribe の会議要約は、ローカルで動作する Ollama を HTTP API で利用します。アプリに LLM モデルは同梱せず、ユーザが別途 Ollama をインストール・起動し、要約用モデルを用意します。

## インストール・起動

1. [Ollama 公式](https://ollama.com/) から macOS 用をダウンロードしてインストールする。
2. ターミナルで `ollama serve` を実行するか、アプリから Ollama を起動する（デフォルトで `http://localhost:11434` で待ち受け）。
3. 要約に使うモデルをインストールする（例: `ollama pull llama3.2`）。設定画面の「要約（LLM）」で表示される一覧は、Ollama にインストール済みのモデルのみです。

## アプリが利用する API

| 用途 | エンドポイント | 説明 |
|------|----------------|------|
| 要約モデル一覧 | `GET http://localhost:11434/api/tags` | インストール済みモデル名の取得。設定画面の Picker に表示。 |
| 要約生成 | `POST http://localhost:11434/api/generate` | 文字起こしテキストを送り、要約文を取得。`model`, `prompt`, `stream: false` を指定。 |

要約のプロンプトは「以下の会議の文字起こしを要約し、議題・決定事項・アクションアイテムに整理して出力してください。」に続けて文字起こしを付与しています。

## 未設定・エラー時

- **要約モデル未選択**: 録画処理では要約をスキップし、文字起こしと録画ファイルのみ出力フォルダに保存します。
- **Ollama 未起動**: 設定画面で要約モデル一覧が空になります。要約実行時に接続エラーとなり、エラーメッセージで案内します。
- **タイムアウト**: 要約 API は 60 秒でタイムアウトします。
