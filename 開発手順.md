# 4. 録画文字起こし機能の作成

録画の文字起こし機能を作成してください。

## 4.1 Whisper エンジンの組み込み

- whisper.cpp（または採用する Whisper 実行エンジン）をビルドし、アプリに同梱するバイナリを用意する。
- Xcode プロジェクトに実行ファイル／フレームワークを追加し、バンドルから起動できるようにする。
- 起動パス・引数（入力音声ファイル、出力テキスト等）の仕様を決め、ドキュメントまたはコメントに残す。

## 4.2 モデル選択・ダウンロード UI

- 初回起動時、または文字起こし用モデルが未設定のときに「どの Whisper モデルをダウンロードするか」を選択するダイアログを表示する。
- 選択可能なモデル一覧（例: tiny, base, small, medium, large-v3 等）と保存先（Application Support 配下）を定義する。
- 選択したモデルを非同期でダウンロードし、進捗をユーザに表示する（必要に応じてキャンセル対応）。
- ダウンロード完了後にモデルを「利用可能」として設定に保存する。

## 4.3 TranscriptionService の実装

- `TranscriptionService` プロトコルを定義し、実装クラスで同梱 Whisper バイナリを実行する。
- 入力: 録画済みファイルの URL（または事前に抽出した音声ファイルの URL）。
- 出力: 文字起こしテキスト（String または構造化データ）。
- 音声抽出: 動画から音声を抽出する処理を用意する（FFmpeg 同梱、AVFoundation のみ、など方針に合わせて実装）。
- エラー・タイムアウト・キャンセルの扱いを決め、呼び出し側でハンドリングできるようにする。

## 4.4 設定・永続化

- 選択済み Whisper モデル（識別子やパス）を `UserDefaults` / `SettingsService` で保存する。
- 設定画面で「文字起こしに利用するモデル」を表示・変更できるようにする（要件定義の画面・UI 要件に合わせる）。

## 4.5 パイプラインとの接続

- `RecordingPipeline` で録画終了後に、確定した動画ファイルを `TranscriptionService` に渡す。
- 文字起こし結果を Domain のモデル（例: `RecordingResult`）に格納し、次の要約処理へ渡せる形にする。
- 文字起こし中の状態（進行中・完了・失敗）を UI またはログで分かるようにする（必要に応じて進捗表示）。
